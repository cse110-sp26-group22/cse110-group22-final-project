# User Research

Project Prompt:
>*You Name It: Code Typing Game* - Typing tutors of old are fun and all, but we want to have some real fun while getting our typing to Claude-level speeds.  Ideas like https://keysoffury.com/ show that we can build a game with typing, but what about what we type - code?  Be it &lt;html>, CSS, or JavaScript we want to challenge to jump, avoid falling code, shoot syntax, or whatever makes sense while learning about the syntax we need to write or recognize.  A game is lot more fun than flash cards, but make sure it can be expanded with new packs for all the UNIX commands, Claude syntax, API methods, or whatever syntax it is you might need to learn by playing in the future.  If we are waiting for class or bored, we might not have our laptop handy, so mobile friendliness is a must, and WiFI can't be required!

The prompt implies the user to be a student learning to code, who wants to perfect their understanding of programming languages syntax.

Due to the short timeframe for research, I obtained most of my data via surveys and a few short conversations. Forms are not the ideal data collection tool, so further interviews could be performed as we build and test our first prototype.  

Based on the information collected, we can narrow down the following persona

## Persona: University CS Student

An undergraduate student who knows the basics of programming languages and is gradually moving from uncertainty to using their skills for projects or research.

They enjoy coding and mostly feel as if they are on the right path.

They have been exposed to a variety of programming languages and their syntax, mostly through university classes. Because of that they are familiar with languages such as Java, C, C++, HTML, CSS, JavaScript, TypeScript, and Python. They are less familiar but nonetheless aware of languages such as SQL or Rust, and commonly used structures and/or protocols such as JSON, APIs, RL environments, and AI syntax.

Outside of studying, they spend their free time listening to music and/or podcasts, watching videos, gaming, and engaging with social media.

### Goals
- Wants to apply their skills to building things they actually want to build
- Wants to pass their classes without stress
- Wants to get an internship

### Pain Points
- I know what I want to do but can't remember the exact syntax
- I get stuck and don't know where to start
- I feel slow compared to classmates or people I see online
- Feels as if they often look up syntax they "should" already know

While they feel slow, most type at above average speed and don't believe that typing speed limits their productivity.

### Motivations
- Find a job
- Build "real" projects
- Personal interest and self improvement

## Stories

As an undergrad student, I want to improve my syntax recollection so I can code without getting stuck.

As someone who is passionate about programming, I want to gain more skills so I don't struggle with starting a new project.

As an undergrad student, I want to solidify my knowledge so I can pass my classes.

## Conclusion

Based on my research syntax recall is more of a problem than typing speed; they might feel slow because they often have to pause to retrieve the correct syntax.

The user is past beginner, but not completely fluent yet. They have breadth but lack depth and confidence.
