## Question/Prompt Research

1. **Type the Code Exactly as Shown:** The app can show users a short piece of HTML, CSS, JavaScript, or UNIX command syntax and ask them to type it perfectly before time runs out. For example, the prompt could show `<button onclick="startGame()">Play</button>` and the user would need to type it exactly. This format would help users practice symbols, brackets, quotation marks, semicolons, and capitalization.

2. **Fill in the Missing Code:** The app can show a code snippet with one missing part and ask the user to type the correct answer. For example, it could show `document.______("title").innerText = "Hello";` and the user would type `getElementById`. This format teaches users how to recognize and complete important syntax without making every question too long.

3. **Choose the Correct Syntax:** The app can ask users to choose the correct version of a code snippet from several options. For example, the question could ask, “Which one correctly links a CSS file to an HTML page?” and the user would select `<link rel="stylesheet" href="style.css">`. This format works well on mobile because users can quickly tap the correct answer.

4. **Fix the Bug:** The app can show broken code and ask the user to type the corrected version. For example, it could show `console.log("Hello World)` and the user would fix it by typing `console.log("Hello World");`. This format is useful because programmers often need to find and correct small mistakes in real code.

5. **Match the Code to Its Purpose:** The app can show a piece of code and ask what it does. For example, it could show `display: flex;` and ask, “What does this CSS property help with?” The user could choose an answer like “layout alignment.” This format helps users understand the meaning of code instead of only memorizing it.

6. **Code Survival Mode:** The game can show falling pieces of code, and the user must type the correct answer before the code reaches the bottom of the screen. For example, the prompt could ask, “Type the JavaScript keyword used to create a variable that can change,” and the correct answer would be `let`. This format makes the learning experience feel more like an arcade game.

7. **Complete the Command:** For UNIX or terminal practice, the app can ask users to complete a command. For example, it could ask, “Type the command to list files in a folder,” and the answer would be `ls`. A harder version could ask, “Type the command to list all files, including hidden files,” and the answer would be `ls -a`.

8. **Identify the Language:** The app can show a code snippet and ask the user whether it is HTML, CSS, JavaScript, UNIX, Python, or another syntax pack. For example, if the app shows `body { background-color: black; }`, the user would answer CSS. This is useful for beginners who are still learning how different coding languages look.

9. **Type the Output:** The app can show a small code snippet and ask the user what the output would be. For example, it could show `console.log(2 + "2");` and the user would type `22`. This format teaches logic, data types, and common programming behavior.

10. **Arrange the Code in Order:** The app can give users scrambled lines of code and ask them to place the lines in the correct order. For example, the user may need to arrange an HTML structure starting with `<!DOCTYPE html>`, then `<html>`, then `<head>`, and then `<body>`. This format is helpful for learning structure and organization.

11. **Speed Typing Challenge:** The app can give users short code phrases to type as fast as possible, such as `const`, `function`, `return`, `margin: 0;`, or `git status`. This format focuses on typing speed, accuracy, and familiarity with common programming terms.

12. **Syntax Shooter Mode:** The app can display moving targets with different syntax options, and the user has to shoot or tap the correct one. For example, the prompt might say, “Select the correct way to create a JavaScript function,” and the targets could include both correct and incorrect function formats. This makes multiple-choice questions feel more game-like.

13. **Real-World Task Prompts:** The app can ask users to type code that completes a small real-world task. For example, the prompt could say, “Create a button that says Submit,” and the user would type `<button>Submit</button>`. Another example could be, “Write a CSS rule that makes text red,” and the user would type `color: red;`. This helps users connect syntax to real coding situations.

14. **Error Message Challenge:** The app can show a common error message and ask the user what likely caused it. For example, it could show “Uncaught ReferenceError: x is not defined” and ask the user to choose or type the explanation. This format helps users become better at debugging.

15. **Flashcard Typing Hybrid:** The app can ask simple recall questions, but the user must type the answer instead of only tapping it. For example, the prompt could ask, “What symbol ends most JavaScript statements?” and the user would type `;`. This works like a flashcard, but it is more active because the user has to type the answer.

16. **Code Recognition Prompts:** The app can ask users to recognize whether syntax is valid or invalid. For example, it could show `let name = "Alex";` and `let = name "Alex"` and ask which one is correct. This format is helpful for beginners because they learn what proper code should look like.

17. **Mobile Mini Challenges:** Since the app needs to be mobile friendly, some prompts should be short and easy to answer quickly. For example, the app could ask, “Which command changes folders?” and the user would tap `cd`. Another question could ask, “Which tag creates a link?” and the user would tap `<a>`. These quick questions would be useful when users are waiting for class or playing for a short amount of time.

18. **Offline Practice Packs:** The app can include downloadable practice packs such as HTML Basics, CSS Selectors, JavaScript Functions, Git Commands, UNIX Commands, API Methods, or AI Prompt Syntax. Each pack can include different question formats, such as typing exact syntax, fixing bugs, choosing correct answers, and completing missing parts. This makes the app expandable in the future.

19. **Boss Battle Questions:** At the end of a level, the user can face a harder challenge where they must answer several questions in a row without making mistakes. For example, a boss battle could ask the user to type a full HTML card, fix a CSS bug, and answer a JavaScript output question. This gives the game a stronger sense of progress and achievement.

20. **Combo Accuracy Prompts:** The app can reward users for typing several correct answers in a row. For example, it could show quick prompts like `git add .`, `git commit -m "update"`, and `git push`, and the user would need to type each command correctly. This format teaches workflows instead of only teaching individual commands.
