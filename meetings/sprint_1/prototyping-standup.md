# Prototyping Team Meeting 

Members Attended:    
Members Absent:    

## Goals:
1. Establish core criteria for the prototype
2. Split up tasks clearly according to feature
3. Agree on theme and goals
4. Debrief personal plans
5. Set expectations of workflow

## Discussion points

### Overall plan
- We talked about how our goal is to create a starting database of questions, a starting user input environment and some sort of scoring or progression system that we could overlay with a game down the line
- We discussed use of AI, most of it will be AI generated and reviewed by members of the team

### Cohesiveness concerns
- We debated how we could come together at the end and put together a cohesive end product
- We went back and forth about how to divide responsibilites best to do this
- We debated changing the scope of Subteam 3 to develop a prototype for the UI, but later decided not to go through with this idea and instead focus on core functionality
- Instead, to make sure the UI is cohesive, we decided to make a lo-fi wireframe first prior to continuing

### Clarifying roles
- The final task distribution was that Subteam 1 would create questions and would be in charge of generating the output of question and answer choices to the user
- Subteam 2 would work on the environment the user would interact with
- Subteam 3 would work on how the game will be evaluated and scored along with how the game's end goals will be achieved.
