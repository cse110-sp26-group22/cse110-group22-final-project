# User Testing Part 2

## Key Issues
1. Game moved on before user was finished answering the question.
    - User said they had a question where the answer was [x*2 for in num]
    - It passed them to the next section before they closed the bracket.
2. The answer is already pre-written for the user.
    - The user said there's no reason for them to read the question when the answer is already present.
3. There's too much information in one place for the user.
    - The user described that they mainly need to look at the text to read and write the information, the timer if they were worried of running out of time.
    - No reason for them to not look at the rest of the screen.

## Addressing the Issues
1. Game moved on before user was finished answering the question.
    - Adjust the sentence to make sure the user fully types information before sending it.
2. The answer is already pre-written for the user.
    - We could make it so the "ghost text" is only present on the second or third try, to encourage thinking through the problem.
3. There's too much information in one place for the user.
    - We can adjust information so that everything is spread out more evenly or that things are closer together.
